# notes/

Working notes, decision records, and the pre-reframe legacy.

## Layout

- `adr/` — Architecture / research decision records. One file per decision, numbered.
- `weekly/` — Weekly sync notes (YYYY-MM-DD.md).
- `legacy/` — Pre-reframe MD notes and artifacts preserved verbatim.
- `onboarding_<handle>.md` — New-collaborator onboarding notes.

## Policy

- A decision that touches `docs/01` or `docs/02` requires an ADR here.
- A weekly sync requires a pre-sync note in `weekly/`.
- Nothing in `notes/` is paper-authoritative; `docs/` is. But any decision reflected in `docs/` should trace back to an ADR or a sync note here.
