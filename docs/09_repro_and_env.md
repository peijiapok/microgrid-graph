# 09 — Reproducibility and Environment

> Last revised 2026-04-22. How to set up the project, how to run the smoke test, how to locate artifacts. Keep this current with the code or the collaborator onboarding breaks.

## 1. Minimum dependencies

- Python 3.10+ (existing code tested under 3.12 and 3.13, per bytecode cache).
- `torch` (CPU is enough for the smoke suite).
- `pandapower`.
- `simbench`.
- `numpy`, `pyyaml`, `pytest`.
- For GNN baselines (C5): `torch-geometric` — install after the core pipeline works.

There is currently no pinned requirements file. A `requirements.txt` + `pyproject.toml` belong at the root; creating them is one of the early tasks.

## 2. Setup (local, Linux/macOS)

```
cd /path/to/microgrid-graph
python -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -e .
# Optional extras:
pip install -e ".[simbench]"   # for SimBench feeder loaders
pip install -e ".[dev]"        # for pytest, ruff, mypy
```

The editable install uses the `pyproject.toml` at the repo root (src-layout). After install, `import sg_resilience` works from any directory.

## 3. Smoke test (target: <10 minutes on CPU)

Run:

```
python -m sg_resilience.smoke
```

It does:
1. Loads case33bw via `configs/smoke_case33bw.yaml` through `benchmark_loader.py`.
2. Generates 5 scenarios (3 persistent stress variants + 2 random) over 4 time steps.
3. Trains the compact GraphSAGE controller for 10 epochs on CPU, `hidden_dim=16`, `num_layers=2`.
4. Evaluates on the base scenario and prints the four core metrics.
5. Asserts feasibility exactly: `0 ≤ a_i ≤ d_i` per node and `sum(a) ≤ P` per step.
6. Exits 0 on success, non-zero with a clear diagnostic on failure.

Representative local run (case33bw, CPU, a few seconds):

```
[smoke] scenario=case33bw_public_benchmark nodes=32 steps=4
[smoke] train_scenarios=5 eval_scenarios=1
[smoke] training compact controller (10 epochs, CPU, tiny)
[smoke] final_loss=-4.43
[smoke] evaluating + asserting feasibility
[smoke]   case33bw_public_benchmark
[smoke]     weighted_served_energy: 97.63
[smoke]     unserved_critical_demand: 1.2e-07
[smoke]     switching_count: 0
[smoke]     critical_continuity_ratio: 1.0
[smoke]   feasibility OK (4 steps)
[smoke] PASS
```

If the smoke fails for a new contributor, something is broken at a load-bearing level — not an edge case.

## 4. Where artifacts live

- **Source of truth:** `src/sg_resilience/` (tracked in git).
- **Configs:** `configs/` (tracked; empty today; populate as C2 landing).
- **Docs:** `docs/` (tracked).
- **Paper:** `paper/` (tracked).
- **Tests:** `tests/` (tracked; empty today).
- **Legacy experiment zips:** `results/legacy_zips/` (**not** tracked; see `.gitignore`). These are historical fixtures, not reproducible from current code without their original configs.
- **New results:** `results/` (not tracked); individual runs produce their own subdirectory; final artifacts archived to Zenodo on release.

## 5. Seeds

Every training run must accept `--seed <int>` and log:
- Python / numpy / torch seeds.
- Git commit hash.
- Config hash.

Artifacts under `results/<run_name>/metadata.json` record these. Any artifact without metadata is not citable.

## 6. Scenario and benchmark provenance

- **SimBench** feeders: loaded via the `simbench` package under permissive terms.
- **pandapower** standard cases: distributed with `pandapower`.
- **Synthetic scenarios**: generated by `scenario_generator.py`; parameters versioned in configs.

None of the raw feeder data is redistributed from this repo. `simbench` and `pandapower` are installed as dependencies.

## 7. Regeneration of legacy results

The 40+ zips in `results/legacy_zips/` correspond to the pre-reframe evaluation protocol. They may not reproduce bit-identically under the new topology splits. Treat them as:
- Sanity-check fixtures for the old protocol, consumable by a dedicated regression test.
- Evidence that the existing code trains and evaluates end-to-end.
- Not ground-truth for the paper. The paper's tables come from runs executed under the protocol in `05_experimental_protocol.md`.

## 9. What to do if something doesn't work

1. Check you are on the right branch.
2. Check the editable install picked up your changes: `python -c "import sg_resilience; print(sg_resilience.__file__)"`.
3. Rerun the smoke test. If it fails, open an issue with the full traceback and the commit hash — do not paper over.
